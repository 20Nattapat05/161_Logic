1. จงเขียนถ้อยแถลงต่อไปนี้ ให้อยู่ในรูปแบบสัญลักษณ์

กำหนดให้

  • P: DataFlag มีค่าเป็น Off
  • Q: Error มีค่าเป็น 0
  • R: Sum < 1,000

(a) Error ไม่มีค่าเป็น 0 แต่ DataFlag มีค่าเป็น Off

คำตอบ: ¬Q ∧ P

(b) DataFlag มีค่าเป็น On หรือ Sum ≥ 1,000 อย่างใดอย่างหนึ่ง

คำตอบ: ¬P ⊕ ¬R

(*รูปแบบเทียบเท่า*) (¬P ∧ R) ∨ (P ∧ ¬R)

(c) DataFlag มีค่าเป็น Off หรือไม่ก็ Error มีค่าเป็น 0 จึงเป็นเหตุให้ Sum < 1,000

คำตอบ: (P ∨ Q) → R

2. จากข้อ 1(a, b, c) จงใช้กฎของเดอมอร์แกน เขียนถ้อยแถลงตรงกันข้าม เป็นภาษาแบบ informal

กำหนดให้

  • P: DataFlag มีค่าเป็น Off
  • Q: Error มีค่าเป็น 0
  • R: Sum < 1,000

(a) ตรงกันข้ามของ ¬Q ∧ P

สัญลักษณ์: Q ∨ ¬P

Informal: “Error เป็น 0 หรือ DataFlag ไม่ได้เป็น Off (คือเป็น On)”

(b) ตรงกันข้ามของ ¬P ⊕ ¬R

สัญลักษณ์: (¬P ∧ ¬R) ∨ (P ∧ R)

Informal: “(DataFlag เป็น On และ Sum ≥ 1,000) หรือ (DataFlag เป็น Off และ Sum < 1,000)”

(c) ตรงกันข้ามของ (P ∨ Q) → R

สัญลักษณ์: (P ∨ Q) ∧ ¬R

Informal: “DataFlag เป็น Off หรือ Error เป็น 0 แต่ Sum ไม่ได้น้อยกว่า 1,000 (คือ Sum ≥ 1,000)”

3. จากข้อ 1(c) จงเขียนบทกลับ บทแย้ง และบทผกผัน เป็นภาษาแบบ informal

จากข้อ 1(c): (P ∨ Q) → R

  • P: DataFlag = Off
  • Q: Error = 0
  • R: Sum < 1,000

บทกลับ (Converse)

สัญลักษณ์: R → (P ∨ Q)

Informal: “ถ้า Sum น้อยกว่า 1,000 แล้ว DataFlag เป็น Off หรือ Error เป็น 0”

บทผกผัน (Inverse)

สัญลักษณ์: ¬(P ∨ Q) → ¬R (เท่ากับ (¬P ∧ ¬Q) → ¬R)

Informal: “ถ้า DataFlag ไม่เป็น Off (คือเป็น On) และ Error ไม่เป็น 0 แล้ว Sum ไม่ได้น้อยกว่า 1,000 (คือ Sum ≥ 1,000)”

บทแย้ง (Contrapositive)

สัญลักษณ์: ¬R → ¬(P ∨ Q) (เท่ากับ ¬R → (¬P ∧ ¬Q))

Informal: “ถ้า Sum ไม่ได้น้อยกว่า 1,000 (คือ Sum ≥ 1,000) แล้ว DataFlag ไม่เป็น Off (คือเป็น On) และ Error ไม่เป็น 0”

4. จงใช้กฎของพีชคณิตบูลีน แสดงวิธีพิสูจน์ว่า

(a) ¬(P ∨ ¬Q) ∨ (¬P ∧ ¬Q) ⇔ ¬P

พิสูจน์:

  1. ¬(P ∨ ¬Q) ∨ (¬P ∧ ¬Q)
  2. (¬P ∧ Q) ∨ (¬P ∧ ¬Q) (De Morgan: ¬(P ∨ ¬Q)=¬P ∧ ¬(¬Q)=¬P ∧ Q)
  3. ¬P ∧ (Q ∨ ¬Q) (Distributive)
  4. ¬P ∧ 1 (Complement: Q ∨ ¬Q = 1)
  5. ¬P (Identity: X ∧ 1 = X)

(b) (P→(Q→R)) ↔ ((P∧Q)→R) ⇔ ?

พิสูจน์:

  1. แปลงอิมพลิเคชัน: A→B = ¬A ∨ B
  2. P→(Q→R) = ¬P ∨ (¬Q ∨ R) = ¬P ∨ ¬Q ∨ R
  3. (P∧Q)→R = ¬(P∧Q) ∨ R = (¬P ∨ ¬Q) ∨ R = ¬P ∨ ¬Q ∨ R
  4. ดังนั้นได้ว่า ทั้งสองข้าง “เหมือนกัน” ให้ X = (¬P ∨ ¬Q ∨ R) จะได้ X ↔ X
  5. X ↔ X = (X∧X) ∨ (¬X∧¬X) = X ∨ ¬X = 1

สรุป: (P→(Q→R)) ↔ ((P∧Q)→R) ⇔ 1 (เป็นจริงเสมอ / Tautology)

6. จงหาว่า ใครคือฆาตกร

กำหนดสัญลักษณ์

  • C: ท่านลอร์ดถูกฆ่าตายด้วยเชิงเทียนตีศีรษะ
  • Pz: ท่านลอร์ดตายด้วยการวางยาพิษ
  • W: ภรรยาอยู่ในห้องกินข้าว ณ เวลานั้น
  • S: ซาร่า (สาวใช้) อยู่ในห้องกินข้าว ณ เวลานั้น
  • K: พ่อครัวอยู่ในห้องครัว ณ เวลานั้น
  • H: หัวหน้าคนใช้วางยาพิษให้ท่านลอร์ด
  • D: คนขับรถเป็นคนลงมือฆ่า
  • V: คนเสิร์ฟไวน์เป็นคนฆ่าท่านลอร์ด

แปลงเงื่อนไขจากโจทย์เป็นตรรกศาสตร์

  • (a) C และโดยความหมาย “ตายด้วยเชิงเทียน” ⇒ ¬Pz
  • (b) W ∨ S
  • (c) K → H
  • (d) W → D
  • (e) ¬K → ¬S
  • (f) S → V
  • สมมติฐานที่ใช้เชื่อมเหตุผล: ถ้า “วางยาพิษให้ท่านลอร์ด” ⇒ “ท่านลอร์ดตายด้วยยาพิษ” คือ H → Pz

พิสูจน์ (Derivation)

  1. จาก (a) ได้ ¬Pz
  2. จากสมมติฐาน H → Pz และข้อ 1 ⇒ ¬H (Modus Tollens)
  3. จาก (c) K → H และข้อ 2 (¬H) ⇒ ¬K (Modus Tollens)
  4. จาก (e) ¬K → ¬S และข้อ 3 ⇒ ¬S (Modus Ponens)
  5. จาก (b) W ∨ S และข้อ 4 (¬S) ⇒ W (Disjunctive Syllogism)
  6. จาก (d) W → D และข้อ 5 ⇒ D (Modus Ponens)

สรุป: D เป็นจริง ⇒ คนขับรถเป็นฆาตกร
หมายเหตุ: (f) ไม่ขัดแย้ง เพราะเราได้ ¬S แล้ว จึงไม่จำเป็นต้องได้ V

7. จงเขียนเป็นภาษาแบบ formal

กำหนดสัญลักษณ์ (Predicates) ที่ใช้

  • Odd(x): x เป็นจำนวนคี่
  • EngComp(s): s เป็นนักศึกษาวิศวกรรมคอมพิวเตอร์
  • LearnLogic(s): s ต้องเรียนตรรกศาสตร์
  • Book(b): b เป็นหนังสือ
  • Person(p): p เป็นคน
  • Read(p,b): p อ่านหนังสือ b
  • Color(c): c เป็นสี
  • Animal(a): a เป็นสัตว์
  • HasColor(a,c): สัตว์ a มีสี c

(a) ผลคูณของสองจำนวนคี่ใดๆ จะได้เป็นจำนวนคี่

Formal: ∀x∀y ((Odd(x) ∧ Odd(y)) → Odd(x·y))

(b) นักศึกษาวิศวกรรมคอมพิวเตอร์ทุกคน จะต้องเรียนตรรกศาสตร์

Formal: ∀s (EngComp(s) → LearnLogic(s))

(c) หนังสือบางเล่มถูกอ่านโดยทุกคน

Formal: ∃b (Book(b) ∧ ∀p (Person(p) → Read(p,b)))

(d) ทุกสีเป็นสีที่มีในสัตว์ทุกตัว

Formal: ∀c (Color(c) → ∀a (Animal(a) → HasColor(a,c)))

8. จงเขียนเป็นภาษาแบบ informal

(a) โจทย์:

∀ จำนวนเต็ม a และ b, ถ้า a และ b เป็นเลขคู่ แล้ว a+b เป็นเลขคู่

Informal: “ผลบวกของจำนวนเต็มคู่สองจำนวน จะเป็นจำนวนคู่เสมอ”

(b) โจทย์:

∀ นักศึกษา s, ถ้า s เรียนวิศวกรรมคอมพิวเตอร์ แล้วต้องเรียนอัลกอริทึม

Informal: “นักศึกษาทุกคน ถ้าเรียนวิศวกรรมคอมพิวเตอร์ ก็ต้องเรียนวิชาอัลกอริทึม”

(c) โจทย์:

∀ คน p ∃ หนังสือ b, p อ่าน b

Informal: “ทุกคนมีหนังสืออย่างน้อยหนึ่งเล่มที่เขาอ่าน”

(d) โจทย์:

∃ สี c ∀ สัตว์ a, a มีสี c

Informal: “มีสีอย่างน้อยหนึ่งสีที่สัตว์ทุกตัวมีสีนี้”

9. จากข้อ 8(a, b, c, d) จงเขียนถ้อยแถลงตรงกันข้าม (ปฏิเสธ) เป็นภาษาแบบ informal

(a) โจทย์เดิม:

∀ จำนวนเต็ม a และ b, ถ้า a และ b เป็นเลขคู่ แล้ว a+b เป็นเลขคู่

ตรงกันข้าม (informal):

“มีจำนวนเต็ม a และ b บางคู่ที่เป็นเลขคู่ทั้งคู่ แต่ a+b ไม่เป็นเลขคู่ (คือเป็นเลขคี่)”

(b) โจทย์เดิม:

∀ นักศึกษา s, ถ้า s เรียนวิศวกรรมคอมพิวเตอร์ แล้วต้องเรียนอัลกอริทึม

ตรงกันข้าม (informal):

“มีนักศึกษาบางคนที่เรียนวิศวกรรมคอมพิวเตอร์ แต่ไม่ได้เรียนวิชาอัลกอริทึม”

(c) โจทย์เดิม:

∀ คน p ∃ หนังสือ b, p อ่าน b

ตรงกันข้าม (informal):

“มีคนอย่างน้อยหนึ่งคนที่ไม่มีหนังสือสักเล่มที่เขาอ่าน”

(d) โจทย์เดิม:

∃ สี c ∀ สัตว์ a, a มีสี c

ตรงกันข้าม (informal):

“สำหรับทุกสี จะมีสัตว์อย่างน้อยหนึ่งตัวที่ไม่มีสีนั้น (ไม่มีสีใดที่สัตว์ทุกตัวมีร่วมกัน)”

10. จงใช้กฎการลงความเห็น พิจารณาว่าข้อโต้แย้งต่อไปนี้ สมเหตุสมผลหรือไม่

(a) โจทย์:

นักศึกษาทุกคน ถ้าเรียนตรรกศาสตร์แล้วจะเก่งคอมพิวเตอร์

สมหญิงเรียนตรรกศาสตร์

ดังนั้น สมหญิงเก่งคอมพิวเตอร์

ให้ S(x): x เป็นนักศึกษา, L(x): x เรียนตรรกศาสตร์, C(x): x เก่งคอมพิวเตอร์

Formal: ∀x (S(x) → (L(x) → C(x))), L(สมหญิง)

สรุปที่ต้องการ: C(สมหญิง)

พิจารณา: จาก L(สมหญิง) อย่างเดียว ยังสรุป C(สมหญิง) ไม่ได้ เว้นแต่รู้ว่า S(สมหญิง) ด้วย

ผล: ไม่สมเหตุสมผล (ข้อมูลไม่พอ / ต้องมี S(สมหญิง))

(b) โจทย์:

ทุกคนที่มีงานทำ จะต้องเสียภาษี

สมชายไม่มีงานทำ

ดังนั้น สมชายไม่ต้องเสียภาษี

ให้ J(x): x มีงานทำ, T(x): x ต้องเสียภาษี

Formal: ∀x (J(x) → T(x)), ¬J(สมชาย)

สรุปที่อ้าง: ¬T(สมชาย)

พิจารณา: รูปแบบคือ “ถ้า J แล้ว T; ไม่ J; ดังนั้น ไม่ T” = การปฏิเสธเหตุ (Denying the antecedent) ซึ่งผิด

ผล: ไม่สมเหตุสมผล

(c) โจทย์:

ถ้าค่าใดเป็นจำนวนคู่ แล้วสองเท่าของค่านั้นก็จะเป็นเลขคู่

2n เป็นเลขคู่

เพราะฉะนั้น n เป็นเลขคู่

ให้ E(x): x เป็นเลขคู่

Formal: ∀n (E(n) → E(2n)), E(2n)

สรุปที่อ้าง: E(n)

พิจารณา: รูปแบบคือ “ถ้า E(n) แล้ว E(2n); E(2n); ดังนั้น E(n)” = ยืนยันตาม (Affirming the consequent) ซึ่งผิด

ผล: ไม่สมเหตุสมผล

(d) โจทย์:

ถ้าผลคูณของสองจำนวนใดๆ เป็น 0 แล้วค่าใดค่าหนึ่งต้องเป็น 0

ทั้ง (x-1) และ (x+1) ไม่เป็น 0

เพราะฉะนั้น (x-1)(x+1) ≠ 0

ให้ A: (x-1)=0, B: (x+1)=0

Formal: ((x-1)(x+1)=0) → (A ∨ B), และมี ¬A ∧ ¬B

ใช้บทแย้ง (Contrapositive): ¬(A ∨ B) → ((x-1)(x+1) ≠ 0) และ ¬(A ∨ B) มาจาก ¬A ∧ ¬B

ผล: สมเหตุสมผล (ใช้กฎ Modus Ponens กับบทแย้ง / De Morgan ได้)
